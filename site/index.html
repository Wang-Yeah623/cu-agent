01

为什么是 Cu Agent

别的 agent 把你绑在 IDE 里盯着，它把你解放到手机上。

📱
入口

微信驱动

从手机发条需求就能开项目，异步推进，随时插话纠偏。

🧠
差异化

人在环里

每步自检进度与偏离；小偏离自修，拿不准就在微信问你，而不是闷头乱写。

🖥️
执行

操控编辑器

通过 VS Code 扩展真的动手改文件、跑命令；没插件时退化为本地直写。

🔌
模型

任意大模型

OpenAI 兼容即可：DeepSeek、Ollama、OpenAI… 改一个环境变量就切换。

02

它每一步都
停下来自检，
而不是一次性甩给你。

收到需求 → 拆成有序子任务 → 逐个执行（操控编辑器写代码）→ 每步检测进度与偏离 → 正常就继续，偏了就自修或在微信问你。这套「执行 + 自检」闭环，正是它区别于一次性代码生成器的地方。

03

工程上

不只是 demo——该有的工程都在。

/01

执行循环

状态机驱动，每任务自检进度，超时保护，可续跑。

/02

安全沙箱

文件操作限制在项目目录，拦截 .. 穿越，危险命令需审批。

/03

状态持久化

项目/任务/进度落盘，进程重启不丢。

/04

VS Code 扩展

独立可加载的编辑器插件，WebSocket 接入，真的操控编辑器。
